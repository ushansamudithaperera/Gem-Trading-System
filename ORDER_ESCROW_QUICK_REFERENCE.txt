# Order & Escrow System - Quick Reference Card

## 🔗 Endpoints

### 1. Update Tracking
```
PUT /api/v1/orders/:id/tracking
Authorization: Bearer SELLER_TOKEN
Content-Type: application/json

{
  "courierCompany": "FedEx",
  "trackingNumber": "794644298644",
  "status": "in_transit"  // optional
}
```

**Response**: Order with status=SHIPPED, autoReleaseDate set

---

### 2. Release Escrow
```
PUT /api/v1/orders/:id/release-escrow
Authorization: Bearer BUYER_TOKEN
Content-Type: application/json

{}
```

**Response**: Order with status=DELIVERED, escrowStatus=RELEASED, transferId

---

## 📋 Requirements Checklist

- [ ] Order model has escrowStatus enum (HELD, RELEASED, REFUNDED)
- [ ] Order model has trackingInfo with carrier, number, status

- [ ] PUT /orders/:id/release-escrow endpoint created
- [ ] Seller can update tracking (only seller of order)
- [ ] Buyer can release escrow (only buyer of order)
- [ ] Role middleware protects endpoints
- [ ] Ownership middleware checks participant
- [ ] Payment captured and transferred to seller
- [ ] Admin fee retained (5%)
- [ ] Transaction counts incremented
- [ ] Auto-release timer set (3 days)

## 🔐 Security

```
Route Protection:
POST /orders           → authMiddleware + isBuyer
PUT /orders/:id/tracking → authMiddleware + isSeller + checkOrderSeller
PUT /orders/:id/release-escrow → authMiddleware + isBuyer + checkOrderBuyer
```

---

## 💳 Payment Flow

```
1. Order Created → Payment Intent HELD
2. Seller Ships → PUT /tracking (Order→SHIPPED)
3. Buyer Confirms → PUT /release-escrow
4. Payment Captured → Funds transferred to seller
5. Seller Receives → Within 1-2 business days
```

## 📊 Order States

| Status | Escrow | Next Action | By |
|--------|--------|-------------|-----|
| PENDING_DISPATCH | HELD | Ship & track | Seller |
| SHIPPED | HELD | Confirm receipt | Buyer |
| DELIVERED | RELEASED | Complete | System |

---

## 🚨 Common Errors

| Code | Message | Fix |
|------|---------|-----|
| 400 | Order not in PENDING_DISPATCH | Already shipped |
| 400 | Order not in SHIPPED status | Seller must ship first |
| 403 | Not authorized | Wrong user/role |
| 404 | Order not found | Check order ID |
| 500 | Failed to transfer funds | Check Stripe account |

## ✅ Validation Rules

**Tracking Update**:
- courierCompany: Required, non-empty string
- trackingNumber: Required, non-empty string
- status: Optional, one of: pending|in_transit|delivered|failed

**Escrow Release**:
- No parameters required
- Only conditions: Order SHIPPED, Escrow HELD, Seller has Stripe account
